# RDR2 Mod Manager SEO Keyword Map

This page documents the SEO strategy for the **RDR2 Mod Manager** repository and GitHub Pages documentation site.

## Primary SEO goal

Rank useful documentation pages for people searching for safe ways to install, organize, backup, disable, and troubleshoot **Red Dead Redemption 2 PC mods**.

## Core keyword cluster

Use these phrases naturally in headings, introductions, alt text, examples, and internal links:

- RDR2 Mod Manager
- Red Dead Redemption 2 mod manager
- RDR2 mods
- install RDR2 mods
- RDR2 LML mod manager
- RDR2 ASI plugin manager
- ScriptHookRDR2 mods
- backup RDR2 mods
- RDR2 mods not loading
- Red Dead Redemption 2 mod profiles
- RDR2 mod conflict detection
- uninstall RDR2 mods safely

## Title pattern

Use titles that combine the product name with a clear user problem:

```text
RDR2 Mods Not Loading: Troubleshooting Guide
Backup and Restore Red Dead Redemption 2 Mods
RDR2 Mod Manager Installation Guide
```

## Description pattern

Every page description should explain what the user gets, not just repeat keywords:

```text
Fix common Red Dead Redemption 2 mod problems including RDR2 mods not loading, crashes, missing LML folders, ASI plugin issues, and broken backups.
```

## Internal linking strategy

Each documentation page should link to related guides. This helps users and search engines understand the relationship between pages:

- Installation → Quick start → Backup
- Troubleshooting → Backup → FAQ
- Compatibility → Supported mods → Glossary
- Architecture → Performance → Roadmap

## What not to do

Avoid:

- keyword stuffing;
- fake download buttons;
- promising support for mods that have not been tested;
- copying third-party mod descriptions;
- hiding affiliation disclaimers;
- using misleading titles like “official RDR2 mod manager”.

## Future SEO improvements

- Add real screenshots with descriptive alt text.
- Add release pages with changelog summaries.
- Add Russian documentation pages under `/ru/` for queries like “менеджер модов RDR2”.
- Add troubleshooting pages for specific error messages when they are known.
- Add a comparison page only if it is fair, factual, and useful.
